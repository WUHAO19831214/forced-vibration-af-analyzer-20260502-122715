# 验证报告与待办

## 当前结论

仓库当前没有提交下列证据：

- 经溯源标定的位移参考数据；
- 实测驱动频率对照；
- 摄像头实测帧率；
- 自动追踪与人工逐帧标点对照；
- 重复性、再现性和不确定度预算；
- 理论模型拟合及残差；
- FFT、相位差或滤波验证。

因此当前只能认定为**功能可运行的教学/探索性原型**，不能给出测量准确度或物理参数可信度结论。

## 建议验证任务

### V1 静态定位

不少于 10 个已知位置，每点 100 帧，报告偏差、标准差、丢失率和误识别率。

### V2 空间标定

使用 0.5、1、2、5 cm 标准长度，在画面中心和边缘验证比例及镜头畸变。

### V3 动态范围

用已知峰—峰位移的平台，在多个频率/幅度下比较 `max(y)-min(y)`，报告 bias、MAE、RMSE 和置信区间。

### V4 频率准确度

使用示波器、频率计或传感器比较 Web Audio 设定频率和执行器实际频率。

### V5 人工视频分析对照

两名分析者盲法标注至少 30 个窗口，比较自动与人工范围、耗时和失败类型。

### V6 理论模型

在驱动力幅值可测且光路放大率已知时，把光斑范围反演为本体单边振幅，再拟合受迫振动模型并报告残差。

## 建议验收指标模板

具体阈值应由研究目的预先规定，例如：

- 静态定位 RMSE ≤ ___ pixel；
- 标尺中心区相对误差 ≤ ___%；
- 动态范围相对误差 ≤ ___%；
- 频率相对误差 ≤ ___%；
- 光斑丢失率 ≤ ___%；
- 人工对照 ICC ≥ ___。

在真实数据完成前不应填入推测数值。
