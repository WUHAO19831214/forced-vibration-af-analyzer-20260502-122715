# 受迫振动振幅与驱动力频率关系分析系统

在线演示：https://wuhao19831214.github.io/forced-vibration-af-analyzer-20260502-122715/

## 项目定位

这是一个浏览器端物理实验原型：Web Audio API 生成正弦信号，实时摄像头追踪红色光斑，用户完成 1 cm 空间标定后，系统把光斑纵向位置范围与当前设定频率配对，并绘制频率响应散点和视觉平滑曲线。

## 科学术语警告

旧版界面把纵轴写作“振幅 A / cm”，但程序实际计算的是记录窗口内：

```text
range_pixel = max(y) - min(y)
range_cm = range_pixel × cm_per_pixel
```

因此当前实现直接得到的是**光斑纵向峰—峰位移范围**，不是由平衡位置到最大位移定义的机械振幅 `A`。若运动近似对称且要从峰—峰范围估计光斑振幅，至少需要除以 2；若要得到振动物体本体振幅，还需建立反射光路放大率模型。程序均未实现这些步骤。

仓库保留历史程序不变，文档用“界面字段 amplitude/旧称 A”和“实际计算量光斑位移范围”区分两者。

## 输入信号来源

### 驱动频率

浏览器 `OscillatorNode` 生成正弦波，程序频率来自当前 `AudioParam.frequency` 设定值。支持：

- 精确数值输入：0–1000 Hz；
- 滑块：1–1000 Hz；
- 自动扫频：默认 0–20 Hz、步进 1 Hz、每步 5 s；
- 音量：GainNode，界面默认增益 0.08。

设定频率不是独立测得的执行器实际频率。普通扬声器在低频、尤其接近 0 Hz 时不能可靠输出相应机械驱动力。

### 位移输入

位移信息来自 `getUserMedia()` 获取的**实时摄像头视频**。当前不支持导入图片、预录视频或传感器文件。

## 位移如何获得

程序逐帧读取 Canvas 像素，将 RGB 转换为 HSV，并通过固定红色阈值筛选候选像素：

- `h ≤ 18°` 或 `h ≥ 340°`；
- `s > 0.38`；
- `v > 0.35`；
- `r > 135`、`r-g > 35`、`r-b > 20`。

候选像素按颜色强度加权，计算红色区域重心 `(x_c, y_c)`。总权重超过 900 时判定锁定。视频宽度超过 1000 px 时按 2 px 步长采样。

这是固定阈值加权重心算法，不是神经网络、连通域跟踪或 OpenCV 模型。

## 像素—厘米标定

用户在视频上绘制实际 1 cm 线段。映射回原始视频后得到 `L_pixel`：

```text
cm_per_pixel = 1 / L_pixel
range_cm = (max(y)-min(y)) × cm_per_pixel
```

标尺与光斑必须尽量共面，摄像头或装置移动后应重新标定。

## 数据记录与采样频率

旧版提供两种记录方式：

1. “记录当前振幅”：把当前窗口范围写入表格，频率字段为空/显示为“视频”；
2. “同步记录”：按下拉菜单设定的 0.2、0.4、0.6、0.8 或 1.0 s 间隔，记录当前频率、范围和页面相对时间。

必须区分两种“采样”：

- **视频追踪采样**：由摄像头帧率和 `requestAnimationFrame` 决定，不是固定值；
- **表格记录间隔**：用户选择的 0.2–1.0 s，仅决定多久结算并写一行。

程序未读取摄像头硬件时间戳，也未验证真实帧率。

## 驱动频率如何获得

表格频率直接读取 `state.currentFrequency`，即程序设定值。它没有从麦克风、示波器、编码器或 FFT 中估计。因此论文中应写“设定频率”，除非另有独立设备验证。

## 共振曲线如何生成

程序从表格筛选数值型 `(frequency, amplitude)`，按频率排序：

- 橙色圆点：原始表格散点；
- 蓝色曲线：用相邻点构造的 Catmull–Rom 风格三次贝塞尔连接；
- 坐标轴范围：根据数据使用 1/2/5/10 数列自动取整。

这不是洛伦兹曲线拟合，也没有拟合质量、参数置信区间或自动共振峰检测。曲线中间值不能当作新增测量数据。

## 适用频率范围

界面允许较宽频率输入，但物理适用范围受以下条件共同限制：

- 执行器实际带宽和输出能力；
- 摄像头帧率和曝光时间；
- 每个记录窗口包含的振动周期数；
- 光斑拖影和极值漏采；
- 浏览器主线程处理能力。

不能把界面上限 1000 Hz 理解为系统已经能可靠测量 1000 Hz 位移。普通 30/60 fps 摄像头无法直接恢复如此高频的逐周期极值。

## 主要误差与边界

颜色误识别、光斑过曝/拖影、摄像头帧率、标尺不共面、透视、光路放大、频率切换瞬态、执行器带宽、浏览器计时抖动和缺少滤波都会影响结果。未经标定和重复验证，本项目应视为教学/探索性原型，而不是计量仪器。
